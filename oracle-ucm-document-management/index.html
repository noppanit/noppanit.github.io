Oracle UCM (Document Management)

May 27th, 2008 12:00 am

Well, actually I have another 4 drafts left, but I will start to write this one first because I want to write it down before I forget it. This topid is about Oracle Content Management??which I was trained a week ago. Interestingly, when I was working at the first company, I had to implement our own Document Management System. So, I pretty have experience based on this product.

A week ago, I was sent to train about UCM, Universal Content Management, which has another 5-6 features to be added on UCM. I was trained about Document Management.

The UCM can handle loads of document. They can be classified into catogory. They can be checked in, or out. It’s like keeping record of every version user check-in or out the document. And the most improtance part of Document Management is searching. While you want to keep your documents into a FileServer and you cannot retrieve them at all. So, searching is vital feature. UCM comes up with full-texted search and it can be changed to be metadata search, depend on your requirements.

The basic is, user check-in the document and it will be stored on the server and secured. When another user wants to check-out the document, which he/she has to have the right permission to deal with the document. He/She can check-out the document, which UCM will keep track on every document, versioning.

I have to say that the most interesting part of UCM is IRM, Information Rights Management. When a user check-out a document. How the company can keep track on that document, because that user can do anything on that document, print it, edit it, etc. So, IRM comes up to help the document when it is off the UCM server.

This topic is briefly.
