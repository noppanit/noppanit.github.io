Tutorial : Having Some Trouble With Prototype and jQuery?

Apr 14th, 2010 12:00 am

Am I the only one who has a problem when using both jQuery and Prototype. You would think why would I want to use those. Why not just use one framework. The reason is some plug-in is using jQuery, and some is using Prototype.

The problem is when I use both frameworks. They are conflict each other, because jQuery also uses $(”) and Prototype also uses $(”). And they cannot be used in the same namespace. So, how are we going to fix it?

Luckily, in jQuery, it has a way to fix it already. We use method jQuery.noConflict(); and whenever you want to use $(”) you use jQuery(‘element’) instead. Simple as that.

1
2
3
4
5
jQuery.noConflict();
// Do something with jQuery
jQuery("div p").hide();
// Do something with another library's $()
$("content").style.display = 'none';
